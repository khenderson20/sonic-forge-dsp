# SonicForge DSP — WebAssembly & 3D Visualization

Real-time audio synthesis and 3D wavetable visualization powered by **SonicForge DSP**, compiled to WebAssembly and rendered in the browser using **Three.js**.

```
┌──────────────────────────────────────────────────────────┐
│                    Browser (Main Thread)                  │
│                                                          │
│  ┌────────────┐   postMessage    ┌──────────────────┐    │
│  │  Three.js  │◄─────────────────│  AudioWorklet     │    │
│  │  (3D Viz)  │   waveform data  │  (audio thread)   │    │
│  └────────────┘                  │                   │    │
│                                  │  ┌─────────────┐  │    │
│                                  │  │ Wasm Module │  │    │
│                                  │  │ SonicForge  │  │    │
│                                  │  │ Oscillator  │  │    │
│                                  │  └─────────────┘  │    │
│                                  └────────┬──────────    │
│                                           │               │
│                                  ┌────────▼──────────┐    │
│                                  │  Audio Output     │    │
│                                  │  (Speakers)       │    │
│                                  └───────────────────┘    │
└──────────────────────────────────────────────────────────┘
```

## ✨ Features

*   **WebAssembly DSP**: C++ oscillator core compiled to raw `.wasm` binary using Emscripten (`STANDALONE_WASM`).
*   **AudioWorklet Integration**: Runs in a separate high-priority audio thread for glitch-free playback.
*   **Zero-Copy Transfer**: Wasm binary is transferred to the worklet via `ArrayBuffer` transferables.
*   **3D Wavetable Visualization**: Real-time 3D rendering of waveform history using Three.js.
*   **Decoupled Animation**: Slow-motion visualization mode that decouples audio rate from render rate, allowing detailed inspection of signal cycles.
*   **Interactive Controls**: Adjust frequency, waveform type, amplitude decay, and additive harmonics in real-time.

## 🛠️ Prerequisites

| Tool | Minimum Version | Purpose |
|------|----------------|---------|
| **Emscripten SDK** | 3.1.50+ | C++ → WebAssembly compiler |
| **CMake** | 3.15+ | Build system |
| **Modern Browser** | Chrome 66+, Firefox 76+, Safari 14.1+ | AudioWorklet + WebAssembly support |

### Install Emscripten

```bash
git clone https://github.com/emscripten-core/emsdk.git
cd emsdk
./emsdk install latest
./emsdk activate latest
source ./emsdk_env.sh
```

## 🚀 Build

### Option 1: From the project root (recommended)

Enable the web build via the CMake option:

```bash
mkdir build && cd build
cmake .. -DSONICFORGE_BUILD_WEB=ON -DCMAKE_BUILD_TYPE=Release
make -j$(nproc)
```

### Option 2: Standalone Emscripten build

```bash
cd web
emcmake cmake -B build-web -DCMAKE_BUILD_TYPE=Release
cmake --build build-web
```

### Build Output

The build produces `sonicforge.wasm` directly into `web/public/`. No Emscripten JavaScript glue code is used; the module is a standalone Wasm binary that is manually instantiated inside the AudioWorklet.

## ▶️ Run

Serve the `public/` directory with a static HTTP server:

```bash
# From the web/ directory
cd ..
python3 -m http.server 8080 --directory public
```

Open **http://localhost:8080** in your browser and click **Start Audio**.

## 🎨 Visualization Features

### 3D Wavetable View
The 3D canvas displays a "ribbon" of waveform slices along the Z-axis.
*   **X-Axis**: Signal Phase (0 → 2π)
*   **Y-Axis**: Amplitude (–1.0 to +1.0)
*   **Z-Axis**: Wavetable Position (History depth)

### Controls
*   **Frequency**: 40Hz – 1200Hz sine/saw/square/triangle wave generation.
*   **Decay**: Linear amplitude envelope applied across the wavetable depth (0% = flat, 100% = full decay).
*   **Harmonics**: Additive synthesis control (1–16 harmonics) to visualize spectral complexity.
*   **Speed**: Controls the "slow-motion" update rate of the 3D animation.

### Slow-Motion Mode
The audio thread runs at full speed (~187Hz), but the visualization updates at a decoupled rate (configurable via the **Speed** slider). This allows you to watch the waveform propagate through the 3D space without motion blur or jitter.

##  Troubleshooting

*   **No Sound**: Ensure you clicked "Start Audio" (browsers require a user gesture). Check that Brave Shields or ad blockers are disabled for `localhost`.
*   **Wasm Fetch Failed**: Ensure `sonicforge.wasm` is in the same directory as `index.html` and served by the HTTP server.
*   **AudioWorklet Error**: Verify your browser supports `AudioContext.audioWorklet`. Safari requires version 14.1+.
*   **Visual Glitches**: If the ribbon looks disconnected, ensure your browser supports `LineSegments` and vertex colors (standard in all modern browsers).
